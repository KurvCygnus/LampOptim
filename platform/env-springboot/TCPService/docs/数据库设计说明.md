# 智绿云控 - AI协同的环境监测与智能路灯节能平台 - 数据库设计说明

以下是根据文中内容整理的关于数据库 `light` 中各个表结构的表格描述：

### 1. 路灯信息表（Device）
| 字段名 | 类型 | 约束 | 默认值 | 描述 |
| :--: | :--: | :--: | :--: | :--: |
| id | VARCHAR(20) | PRIMARY KEY |  | 路灯编号[主键] |
| ip | CHAR(20) | NOT NULL | '0.0.0.0' | 路灯ip地址，默认为一个无效IP地址或根据实际需求设置 |
| auto | INT | NOT NULL | 0 | 自动调节开关，默认为0（可能表示关闭自动调节） |
| brightness | INT | NOT NULL | 50 | 亮度，默认为50（或根据实际需求设置） |
| port | INT |  | NULL | 端口(暂时未使用)，默认为NULL |
| lightValue | INT |  | 0 | 感光器值，默认为0 |
| lightStatus | INT | NOT NULL | 0 | 灯状态(开关)，默认为0（可能表示关闭） |
| isBroken | INT | NOT NULL | 0 | 是否损坏，默认为0（表示未损坏） |
| lastTime | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 上次操作时间，默认为当前时间戳 |
| onLine | INT | NOT NULL | 1 | 是否在线，默认为1（表示在线） |
| isBusy | INT | NOT NULL | 0 | 是否忙碌，默认为0（表示不忙碌） |
| INDEX (ip) |  |  |  | 为ip地址添加索引，提高查询效率（可选） |

### 2. 用户表（User）
| 字段名 | 类型 | 约束 | 默认值 | 描述 |
| :--: | :--: | :--: | :--: | :--: |
| userId | INT | AUTO_INCREMENT PRIMARY KEY |  | 用户id |
| username | VARCHAR(20) | UNIQUE NOT NULL |  | 用户名(唯一) |
| password | VARCHAR(20) | NOT NULL |  | 密码 |
| userPermission | INT | NOT NULL |  | 权限 |

### 3. 维修日志表（MaintenanceLog）
| 字段名 | 类型 | 约束 | 默认值 | 描述 |
| :--: | :--: | :--: | :--: | :--: |
| id | INT | AUTO_INCREMENT PRIMARY KEY |  | 日志id[主键自增] |
| time | DATETIME | NOT NULL |  | 时间 |
| subject | VARCHAR(40) | NOT NULL |  | 主题(20字以内) |
| content | VARCHAR(1000) | NOT NULL |  | 日志内容(文本,1000字以内) |
| userId | INT |  |  | 维修人员id(外键) |
| FOREIGN KEY (userId) |  | REFERENCES User(userId) |  | 外键约束，引用User表的id字段 |
| INDEX (userId) |  |  |  | 为维修日志表的userId字段添加索引，提高查询效率（可选） |

### 4. AI员工表（ai_employee）
| 字段名 | 类型 | 约束 | 默认值 | 描述 |
| :--: | :--: | :--: | :--: | :--: |
| ai_id | BIGINT | NOT NULL AUTO_INCREMENT PRIMARY KEY |  | AI员工ID[主键自增] |
| name | VARCHAR(100) | NOT NULL |  | 姓名 |
| job_description | TEXT | NOT NULL |  | 职位描述 |
| work_start_time | TIME | NOT NULL |  | 工作开始时间 |
| work_end_time | DATETIME |  | NULL | 工作结束时间 |
| daily_summary | TEXT |  | NULL | 每日总结 |
| user_id | BIGINT |  | NULL | 关联用户ID |
| is_working | TINYINT | NOT NULL | 0 | 是否工作中（0:否 1:是） |

### 5. AI操作日志表（ai_action_log）
| 字段名 | 类型 | 约束 | 默认值 | 描述 |
| :--: | :--: | :--: | :--: | :--: |
| action_id | BIGINT | NOT NULL AUTO_INCREMENT PRIMARY KEY |  | 操作日志ID[主键自增] |
| ai_id | BIGINT |  | NULL | AI员工ID |
| action_description | TEXT | NOT NULL |  | 操作描述 |
| action_time | DATETIME | NOT NULL |  | 操作时间 |
| INDEX (ai_id) |  |  |  | 为ai_id添加索引 |

这些表格描述了 `light` 数据库中各个表的结构，包括字段名、类型、约束、默认值以及描述。
